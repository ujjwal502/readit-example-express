## Answers & explanations

**Q1 — B.** Every routing method forwards to the lazily-created base router; `app.get('/x', h)` desugars to `this.router.route('/x').get(h)` (`lib/application.js:246`, `:502`). Matching lives in the `router` dependency.

**Q2 — B.** `app.use` flattens arguments with `flatten.call(..., Infinity)` (`lib/application.js:193`), so `[[fnA]], fnB` → `[fnA, fnB]` → 2 registered.

**Q3.** It returns the `view engine` **setting** value. The verb generator special-cases GET: `if (method === 'get' && arguments.length === 1) return this.set(path)` (`lib/application.js:503`) — one argument means no handler, so it's a settings read, not a route.

**Q4.** A `use()` with no function is almost always a bug (typo / undefined import). Silently ignoring it would surface later as "my middleware never runs"; throwing at the call site (`lib/application.js:196`) fails fast where the mistake is. *(Inference on rationale.)*

**Q5 — C.** `app.handle` installs `finalhandler` as the default `done` (`lib/application.js:146`); an exhausted stack calls `done()` → 404.

**Q6 — B.** `exports.methods = METHODS.map(m => m.toLowerCase())` (`lib/utils.js:28`) then `methods.forEach(method => app[method] = …)` (`lib/application.js:498`).
