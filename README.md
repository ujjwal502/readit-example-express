# The Express 5 Codebase Book

A reference-grade guide to the **Express 5.2.1** source in this repository. The goal: after
reading this book, you can answer virtually any design or implementation question about
Express — architecture, data flow, contracts, edge cases, failure modes — **without going back
to search the source.**

Express is a "fast, unopinionated, minimalist web framework for Node.js" (`package.json:3`).
Its own code is remarkably small — **6 files, ~2,765 lines in `lib/`** — because Express 5 is a
thin **decoration-and-dispatch layer** over `node:http` that delegates routing, body parsing,
static serving, and more to focused npm packages. This book covers both Express's own code (in
depth) and the delegated behavior (at contract level, grounded in the in-repo test suite).

> **Grounding note.** This checkout has **no `node_modules`**, so external packages (`router`,
> `body-parser`, `serve-static`, `send`, …) can't be read directly. Their **contracts** are
> grounded in the repo's 75-file test suite — which exercises them as executable specs — plus
> how `lib/` uses them and what `History.md` records. Anywhere the mechanism lives in an
> external package, the book says so explicitly and cites the pinning tests.

## How this book is grounded

- **Every non-trivial claim cites `file:line`.** Claims about Express's own behavior cite
  `lib/*.js`; claims about delegated behavior cite the pinning `test/*.js`.
- **Inferences are labeled.** Where the code doesn't fully determine behavior, the text says
  "(inference)" or lists it under "Open questions."
- **External-package internals are out of scope** but their contracts are documented and pinned
  to the versions in `package.json`.
